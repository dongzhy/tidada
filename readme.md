# MBTI 性格测评小程序
基于 Taro3 + React + TypeScript + Sass 开发的轻量化 MBTI 性格测试微信小程序，支持快速答题、自动计算性格类型、展示测评结果，界面简洁、体验流畅。

## 项目简介
`mbti-mini-test` 是一款轻量型微信小程序，内置完整 MBTI 测试题库，用户完成答题后自动计算性格维度，展示 16 型人格结果与详细描述。
- 纯前端实现，无需后端接口
- 完整答题 + 结果计算 + 结果展示业务闭环
- 支持答案本地缓存、全流程异常容错处理
- 基于 Taro UI 组件库快速搭建界面

## 技术栈
- 框架：Taro 3.6.28
- 开发语言：React 18 + TypeScript
- 样式：Sass
- UI 组件：Taro UI
- 构建工具：Webpack5
- 运行平台：微信小程序（支持一键编译多端）

## 核心功能
### 答题模块
- 内置 10 道标准 MBTI 性格测试题
- 覆盖 E/I、S/N、T/F、J/P 四大性格维度
- 自动统计用户各维度答案得分
- 根据得分匹配对应 16 型人格结果

### 结果展示模块
- 展示人格类型名称、完整详细解读
- 全流程异常捕获、空值容错处理
- 答案本地持久化存储，刷新页面不丢失答题记录
- 结果页支持一键返回首页

### 多端编译支持
项目内置多端构建脚本，可一键编译打包至：支付宝小程序、抖音小程序、QQ小程序、京东小程序、H5、快应用等平台

## 核心页面逻辑说明
### 结果页逻辑
1. 读取本地缓存的用户答题答案列表
2. 校验答案完整性与 JSON 数据格式，非法数据直接拦截
3. 调用 `getBestQuestionResult` 工具方法计算最终人格结果
4. 多层异常捕获：答案为空、数据格式错误、计算逻辑异常统一兜底提示
5. 提供返回主页按钮，支持页面重定向重置答题状态

## 运行流程
进入小程序首页 → 点击开始答题 → 完成全部题目提交答案 → 系统自动计算 MBTI 人格类型 → 展示测评结果（人格名称+完整详细描述）


# TiDaDa AI 答题测评系统
## 项目简介
TiDaDa 是一套前后端分离全栈 AI 答题测评系统，后端基于 SpringBoot 开发，Web 管理后台采用 Vue3 + TypeScript，微信小程序端基于 Taro3 跨端实现。

平台同时支持「自定义计分答题」「AI 智能性格测评」两套业务模式，搭建普通用户 / 管理员两级权限体系，覆盖**应用创建、题目管理、AI 批量出题、评分配置、用户答题、应用审核、数据统计**完整业务链路。

项目地址：https://github.com/dongzhy

## 一、技术栈
### 后端技术栈
- 核心框架：SpringBoot 2.7、SpringMVC、MyBatis-Plus
- 数据库：MySQL（6 张核心业务数据表）
- 缓存 & 并发控制：Caffeine 本地缓存、Redis、Redisson 分布式锁
- AI 能力：智谱大模型 SDK、SSE 流式输出、RxJava 异步流处理
- 权限体系：自定义 `@AuthCheck` 注解实现接口级权限拦截
- 工具类：Hutool、EasyExcel（待开发 Excel 批量导入题库功能）
- 设计模式：策略模式（双评分策略解耦）
- 待集成中间件：RabbitMQ、Nacos、SkyWalking、ELK

### Web 管理后台
- 基础框架：Vue3 + TypeScript + Vue-CLI
- 状态管理：Pinia
- UI 组件库：Arco Design Vue
- 工具库：dayjs、ECharts、qrcode
- 工程规范：ESLint + Prettier、OpenAPI 自动生成接口请求代码

## 二、数据库设计（6 张核心业务表）
1. **user 用户表**
账号、密码、角色、微信 openid、昵称、头像、个人简介
2. **app 应用表**
应用名称、描述、图标、应用类型、评分策略、审核字段（审核状态 / 审核人 / 审核时间 / 驳回理由）
3. **question 题目表**
JSON 存储题目与选项，区分两套数据结构：
- 得分类：选项携带分值
- 测评类：选项携带 MBTI 属性标识
4. **scoring_result 评分结果表**
测评名称、描述、展示图片、属性匹配数组、得分匹配区间
5. **user_answer 用户答题记录表**
用户所选答案、最终得分 / AI 测评结果、关联应用 ID、用户 ID
6. 统计聚合：Mybatis 自定义聚合 SQL，统计答题量、测评结果分布数据

## 三、后端模块 & 接口说明
### AppController 应用模块
1. 新增应用：普通用户创建应用，默认状态为待审核
2. 编辑应用：用户修改自有应用，修改后重置审核状态为待审核
3. 管理员更新：全量编辑应用所有信息，无权限限制
4. 审核接口：管理员审核，支持通过 / 驳回，可填写驳回理由
5. 分页查询三类接口：
   - 管理员端：查询平台全部应用（包含待审核、已驳回）
   - 个人中心：仅查询当前登录用户创建的应用
   - 首页展示：仅展示审核通过的公开应用
6. 权限控制：普通用户仅可操作自身创建应用，管理员拥有全部审核、管理权限

### QuestionController 题目模块
基础 CRUD 能力 + AI 智能出题核心能力
1. 同步 AI 出题：根据应用信息批量生成题目 JSON 数组
2. SSE 流式 AI 出题：长文本逐字符实时推送，消除页面长时间等待阻塞
3. 自动适配两类应用：得分类选项携带分值、测评类携带 MBTI 属性标识

### ScoringResultController 评分结果模块
仅自定义得分类应用需要配置，AI 测评应用无需手动维护评分结果
提供新增、删除、管理员更新、用户编辑、分页列表、详情 VO 接口；自动完成 List 数组与数据库 JSON 字符串序列化转换。

### UserAnswerController 答题核心模块
基于策略模式实现两套独立打分逻辑
1. 自定义计分策略：累加每题选项分数，匹配最低分数区间返回对应测评结果
2. AI 测评策略：本地缓存 + 分布式锁拦截重复 AI 调用，降低大模型调用成本
3. 用户提交答题后自动执行打分逻辑，回填最终结果存入答题记录表

### AppStatisticController 数据统计模块
自定义聚合 SQL，为前端图表提供数据源：
- 热门应用答题数量排行
- 单个应用内各测评结果分布统计

### UserController 用户模块
- 普通用户：账号注册、密码登录、退出登录、获取当前登录用户、修改个人信息
- 管理员：新增用户、分页管理平台全量用户

## 四、Web 管理后台页面功能
1. 首页：展示全部已过审公开应用，支持搜索、生成分享二维码
2. 应用管理（管理员）：全平台应用列表、多条件筛选、批量审核
3. 我的应用：个人创建应用，快速跳转题目配置 / 评分配置页面
4. 题目管理：手动新增题目、AI 一键批量生成题目
5. 评分结果管理：得分类应用配置评分区间（已修复修改按钮点击无响应线上 Bug）
6. 答题记录：查看所有用户完整答题记录
7. 数据统计：答题量、测评结果分布可视化图表
8. 用户管理（管理员）：平台所有用户增删改查
